Description

Memory for large datacenters was originally stored on large disks with laser scanners on them.
The memory would be broken up into sectors containing a number of bytes. To make this more efficient
a File allocation table was created. A FAT is a memory storage structure that uses an index table of corresponding disk
blocks to store files efficiently.They are still used in devices that require memory efficiency over file access speed, like NXP boards.

Technical Concepts

A FAT is technically challenging to implement. Sectors are stored in a scheme where the first sectors
act as indices to blocks. The blocks that are indexed to in the table are then linked to different indices.

In this example the table starts at sector 0. The files can occupy the 9 others.
The first available sector is 1 if the file needs more memory than sector 1 occupies it will proceeed to 1's
link in the FAT. 7 is the next sector that is free. It will continue until either
A. The sectors have adequately stored the file
B. The last sector is used in which case the file cannot be stored and should nothing should be written to any sector.
